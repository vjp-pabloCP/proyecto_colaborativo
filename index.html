Git Bash y GitHub

Configuración de Git Bash

Para comenzar a usar Git Bash, sigue estos pasos:

  1. Descarga e instala Git desde git-scm.com
  2. Abre Git Bash y configura tu usuario: 
    git config --global user.name "Tu Nombre"
    git config --global user.email "tuemail@example.com"

Clonar un repositorio desde GitHub

Para clonar un repositorio usa el siguiente comando:

git clone https://github.com/usuario/repositorio.git

Subir cambios a GitHub

Después de modificar archivos, usa los siguientes comandos:

git add .
git commit -m "Descripción del cambio"
git push origin main

Información sobre GitHub y Git Bash

Git es un sistema de control de versiones distribuido utilizado para rastrear los cambios en los archivos de un proyecto. GitHub, por su parte, es una plataforma basada en la web para alojar y compartir repositorios de Git. Git Bash es una interfaz de línea de comandos que emula una terminal de Linux en Windows, y es comúnmente usada para interactuar con Git y GitHub.

¿Qué es Git?

Git es un sistema de control de versiones que permite a los desarrolladores colaborar de manera eficiente en un proyecto, manteniendo un historial de todos los cambios realizados en los archivos y permitiendo trabajar en paralelo en diferentes funcionalidades o correcciones sin interferir entre sí. Git permite revertir cambios, fusionar diferentes versiones de un archivo y mantener un control detallado de las modificaciones en el proyecto.

¿Qué es GitHub?

GitHub es una plataforma de desarrollo colaborativo basada en la web que permite alojar proyectos Git en repositorios remotos. Además de gestionar proyectos de código, GitHub permite colaborar con otros desarrolladores, gestionar problemas, integrar herramientas de CI/CD (Integración Continua/Despliegue Continuo), realizar revisiones de código y más. GitHub facilita el trabajo en equipo y la contribución a proyectos open source.

¿Qué es Git Bash?

Git Bash es una aplicación para Windows que proporciona una terminal emulada con una serie de comandos comunes de Linux, permitiendo a los usuarios interactuar con Git desde la línea de comandos en un entorno Windows. Con Git Bash, los usuarios pueden ejecutar comandos de Git como si estuvieran en un sistema operativo basado en Unix, lo que facilita su uso en proyectos que utilicen Git para el control de versiones.

Comandos Básicos de Git

  • git init: Inicializa un nuevo repositorio de Git en el directorio actual.
  • git clone URL: Clona un repositorio remoto a tu máquina local.
  • git add archivo: Añade archivos al área de preparación para un commit.
  • git commit -m "mensaje": Guarda los cambios en el repositorio local con un mensaje descriptivo.
  • git push: Envía los cambios locales a un repositorio remoto, como GitHub.
  • git pull: Obtiene y fusiona cambios desde un repositorio remoto.
